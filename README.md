Can wallet behaviour, textual signals, and broader market context identify mispriced Polymarket
trades in geopolitical prediction markets? This project builds a pipeline from 150M+ on-chain
`OrderFilled` events through to a cold-start XGBoost model that ranks informed trades on wallets never seen in training. Held-out ROC-AUC rises from 0.9432 (market baseline) to 0.9534 on unseen wallets, with the top-decile alpha slice delivering +20.9pp realised edge.

See `report/sentiment_report.pdf` for the full write-up.

## Running the pipeline

The full chain takes several hours on first run. If a teammate has already built the
`polymarket.ddb` file, drop it into `data/analytical/` and skip step 1.

### 1. Build the database (10 minutes first run (depending on internet speed))

```bash
make pipeline
```

S3 download -> markets -> tokens -> filter -> process -> load. Output: `data/analytical/polymarket.ddb`.

### 2. Sentiment + financial enrichment

The three tracks have very different runtimes. Run the cheap ones individually during the day and
leave the FT track to run overnight.

```bash
make sentiment-financial     # -5 minutes - yfinance weekly abnormal returns
make sentiment-ts            # -10 minutes - Trump Truth Social + VADER
make sentiment-ft            # several hours - parse ProQuest, score with FinBERT, aggregate
```

Or run all three end-to-end with `make sentiment` (blocks on the FT step).

All three write to the `sentiment` and `financial` tables in `polymarket.ddb` via
`make load-sentiment` (invoked internally).

### 3. Wallet EDA (Section 3)

```bash
make analysis
```

Or run `notebooks/wallet_analysis.ipynb`.

Produces `figures/wallet_activity.png`, `cum_volume_curve.png`, `hitrate.png`, `vol_hit.png`,
`days_before.png`, `timing_hit.png`, `heatmap.png`, and exports
`data/analytical/smart_wallets.csv`.

### 4. Machine learning (Section 5)

```bash
make ml
```

Or run `notebooks/ml_report.ipynb`.

Produces `figures/cold_start_diagnostics.png`, `cold_start_evaluation.png`,
`cold_start_feature_importance.png`, `risk_adjusted_profile.png`.

Expect -10–20 minutes after the DB and enrichment tables are in place. The first run caches
`data/processed/trades_enriched.parquet`; subsequent iterations skip the heavy DuckDB pull.

### Everything at once

```bash
make all
```

## Configuration

All thresholds live in config modules. Change a constant there; no hunting through module bodies.

| Module | What it controls |
|---|---|
| `src/machine_learning/config.py` | Holdout sizes, Optuna trials, friction parameters, feature-family lists |
| `src/analysis/config.py` | Whale-quadrant thresholds, HHI top-50, cold-start cutoff date |
| `src/sentiment/pipeline_config.py` | Topic -> keyword -> corpus mappings, ticker baskets, stopwords |

## Data sources

| Source | Role |
|---|---|
| `warproxxx/poly_data` S3 archive | 150M+ `OrderFilled` events (1 Feb 2023 - 7 Oct 2025) |
| Polymarket Gamma API | Market metadata (5000+ geopolitical markets) |
| Polymarket CLOB API | ERC-1155 token → condition ID mapping |
| Financial Times via ProQuest | 51,647 articles scored with FinBERT |
| `stiles/trump-truth-social-archive` | 29,410 Trump posts scored with VADER |
| yfinance | Daily OHLCV for geopolitically-linked tickers |

## Known caveats

- First full run is several hours; reuse a teammate's `.ddb` if possible.
- FinBERT scoring is the slowest sentiment step (8+ hours for 51k articles on CPU).
- Clear the parquet cache (`make ml-clean`) after any config change, otherwise the ML pipeline
  will load a stale feature matrix.
- `data/processed/financial_signals.csv` is a daily-price CSV consumed by the ML pipeline
  (separate from `data/processed/financial/financial_signals_weekly.csv`, which is the weekly
  signals loaded into DuckDB by the sentiment pipeline).
